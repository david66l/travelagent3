## 二、详细问题清单

### 2.1 Agent 体系：严重缺失（7/10 缺失）

**v4 设计文档定义的 10 个 Agent：**

| # | Agent 名称 | 技术方案是否实现 | 状态 |
|---|-----------|----------------|------|
| 1 | Intent Recognition Agent | ✅ 已实现 | `intent_recognition.py` |
| 2 | Information Collection Agent | ❌ **缺失** | 功能被 IntentRecognitionAgent 内联 |
| 3 | Real-time Query Agent | ❌ **缺失** | 功能被 Orchestrator 直接调用 Skill 替代 |
| 4 | Preference & Budget Agent | ✅ 已实现 | `preference_budget.py` |
| 5 | Itinerary Planner Agent | ✅ 已实现 | `itinerary_planner.py`（但严重简化） |
| 6 | Q&A Agent | ❌ **缺失** | 功能被 Orchestrator 直接调用 LLM 替代 |
| 7 | Memory Management Agent | ❌ **缺失** | 只有数据模型，无 Agent 实现 |
| 8 | Proposal Generation Agent | ❌ **缺失** | 功能被 Orchestrator._generate_proposal() 内联 |
| 9 | Validation Agent | ❌ **缺失** | 完全未实现 |
| 10 | Map & Route Agent | ❌ **缺失** | 完全未实现 |

**具体问题：**

1. **Information Collection Agent 缺失**
   - v4 中独立 Agent，负责收集和补全用户信息
   - 技术方案中此功能被 IntentRecognitionAgent 和 Orchestrator 内联处理
   - 导致职责不清晰

2. **Realtime Query Agent 缺失**
   - v4 中独立 Agent，内部并行发起多个查询（景点、天气、价格）
   - 技术方案中 Orchestrator 直接调用 `poi_search.search_pois()`，跳过了 Agent 抽象层
   - **违背 v4 原则：LLM 不直接操作数据，通过 Agent 接口**

3. **Q&A Agent 缺失**
   - v4 中独立 Agent，基于 RAG + 网络搜索回答用户问题
   - 技术方案中 `_handle_qa()` 直接调用 LLM，没有使用 RAG
   - 缺少知识库检索能力

4. **Memory Management Agent 缺失**
   - v4 中详细实现了 4 类记忆的管理（对话/行程/偏好/预算）
   - 技术方案中只有 Pydantic 模型定义，没有 Agent 实现代码
   - 没有 `save_conversation_turn()`, `save_itinerary()`, `get_user_memory()` 等核心方法

5. **Proposal Generation Agent 缺失**
   - v4 中独立 Agent，将行程转化为自然语言方案
   - 技术方案中作为 Orchestrator 的私有方法 `_generate_proposal()` 内联实现

6. **Validation Agent 缺失**
   - v4 中独立 Agent，包含预算/时间/POI 真实性/开放时间等多维度校验
   - 技术方案完全未实现
   - **后果：行程质量无保障**

7. **Map & Route Agent 缺失**
   - v4 中独立 Agent，包含地理坐标查询、距离矩阵、2-opt 路线优化
   - 技术方案完全未实现
   - ItineraryPlannerAgent 中只有 `_sort_activities()` 简单排序

## 五、结论

技术方案目前只实现了 v4 设计文档约 **40%** 的内容。核心问题：

1. **Agent 体系不完整**：只实现了 3/10 个 Agent，系统缺少记忆、校验、路线优化等核心能力
2. **Skill 层不完整**：缺少天气、路线、记忆存取等 Skill
3. **记忆系统未落地**：有数据模型但无实际保存逻辑
4. **RAG 完全缺失**：Q&A 能力受限
5. **路线优化缺失**：行程路线是简单排序，不合理
6. **用餐逻辑缺失**：行程缺少用餐安排

**建议**：按上述 4 个阶段逐步补齐，预计需要额外 **2-3 周** 开发时间。
